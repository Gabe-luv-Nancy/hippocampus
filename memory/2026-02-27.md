# 2026-02-27 事故记录

## ⚠️ 重大事故：硅基流动配置导致服务中断

### 事故经过
- 我尝试直接在 models.json 中添加硅基流动模型配置
- **结果：OpenClaw 无法启动，持续好几个小时**
- Gabe 手动还原 models.json 后服务才恢复正常

### 根本原因
- models.json 配置格式敏感，错误配置会导致整个服务崩溃
- 直接修改配置文件风险太高，没有回滚机制

### 解决方案
1. 🚫 **永久禁止**直接修改 models.json
2. ✅ 创建独立脚本调用硅基流动 API
3. ✅ 通过子任务执行脚本，出错不影响主进程
4. ✅ 已创建 `scripts/siliconflow-vlm.sh` 脚本

### 教训
- 配置文件修改 = 高危操作，必须隔离
- 脚本 + 子任务 = 安全可控
- 宁可多一层抽象，不要冒险搞挂自己
