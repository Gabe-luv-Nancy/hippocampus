# 问题与困难总结：TCP抓包方案

## 背景

最初目标：获取东方财富期货软件的实时数据

## 方案：TCP抓包实施步骤

1. 监控 `mainfutures` 进程的网络连接
2. 使用 Wireshark/tshark 抓取进程的网络数据包
3. 解析 TCP 流还原应用层数据

## 遇到的问题

| 问题 | 描述 |
|------|------|
| **依赖外部工具** | 需要安装 Wireshark，引入额外框架 |
| **接口选择困难** | Windows 有多个网络接口（以太网、Wi-Fi、虚拟网卡等） |
| **过滤条件复杂** | 需要猜测端口号、协议类型 |
| **数据解析困难** | TCP 流是二进制数据，需要手动解析应用层协议 |
| **数据格式未知** | 东方财富使用私有协议，需要逆向工程 |
| **不稳定** | 进程 PID 每次启动会变化 |
| **法律风险** | 逆向商业软件可能存在合规问题 |

## 根本问题

**本末倒置** - 为了获取数据，需要：
- 学习复杂的网络抓包技术
- 逆向未知协议
- 依赖第三方工具
- 处理大量无关数据
