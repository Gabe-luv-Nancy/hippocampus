# 核心能力建设：外部AI编程助手集成框架

## 概述

本文档记录我如何通过外部AI编程助手（如Cline）来扩展代码生成能力。这是我的**核心能力**，不仅限于LOTT项目，而是适用于所有工程开发任务。

---

## 一、架构设计

### 1.1 系统架构
```
┌─────────────────────────────────────────────────────────────────┐
│                        我的环境 (WSL Ubuntu)                    │
│  ┌─────────────────┐  ┌─────────────────┐  ┌───────────────┐  │
│  │  主对话模型      │  │  任务规划器      │  │  执行器       │  │

│           │                    │                    │          │
│           └────────────────────┼────────────────────┘          │
│                                ▼                               │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │              外部AI编程助手 (Windows)                    │  │
│  │  ┌─────────────────────────────────────────────────────┐│  │
│  │  │  Cline + GLM-5 (用户配置)                         ││  │
```

### 1.2 核心优势

| 能力 | 说明 |
|------|------|
| **任务分解** | 将复杂需求拆分为独立模块 |
| **外部执行** | 调用Cline执行代码生成，不消耗我的上下文 |
| **结果复核** | 检查生成的代码正确性 |
| **测试验证** | 运行测试确保代码质量 |
| **迭代优化** | 分析失败原因，循环调整 |

## 二、Cline集成

### 2.1 Cline CLI位置
```
C:\Users\{用户名}\AppData\Roaming\npmcline.cmd
```

### 2.2 调用方式
通过PowerShell调用：
```python
POWERSHELL = "/mnt/c/Windows/System32/WindowsPowerShell/v1.0/powershell.exe"

cmd = f'cd /d {workspace}; cline ask "{prompt}"'
subprocess.run([POWERSHELL, "-Command", cmd])
```

### 2.3 ClineRunner脚本
位置: `/home/gabetopz/.openclaw/workspace/cline_runner.py`

```python
from cline_runner import run_cline_task

result = run_cline_task("创建测试用例", timeout=600)
```

## 三、工作流程

### 3.1 完整流程
```
用户需求
    │
    ▼
┌─────────────────────────────┐
│  1. 需求分析                │
│     - 理解需求               │
│     - 技术选型               │
└──────────────┬──────────────┘
               │
    ▼
┌─────────────────────────────┐
│  2. 任务分解                │
│     - 拆分为独立子任务       │
│     - 评估上下文长度         │
│     - 规划执行顺序           │
└──────────────┬──────────────┘
               │
    ▼
┌─────────────────────────────┐
│  3. 生成提示词              │
│     - 详细技术需求          │
│     - 代码规范要求          │
│     - 测试要求              │
└──────────────┬──────────────┘
               │
    ▼ [调用Cline]
┌─────────────────────────────┐
│  4. 执行代码生成            │
│     - Cline调用GLM-5       │
│     - 生成代码到目标目录    │
└──────────────┬──────────────┘
               │
    ▼
┌─────────────────────────────┐
│  5. 结果复核                │
│     - 语法检查              │
│     - 逻辑验证              │
│     - 查找明显错误          │
└──────────────┬──────────────┘
               │
    ▼ (需要测试)
┌─────────────────────────────┐
│  6. 测试验证                │
│     - pytest单元测试        │
│     - 集成测试              │
│     - 人工测试（如需要）    │
└──────────────┬──────────────┘
               │
    ▼ (失败)
┌─────────────────────────────┐
│  7. 迭代优化                │
│     - 分析失败原因          │
│     - 生成修复提示词        │
│     - 新Cline对话执行       │
└─────────────────────────────┘
```

### 3.2 上下文管理策略

| 场景 | 策略 |
|------|------|
| 简单任务 (<50K tokens) | 单个Cline对话完成 |
| 复杂任务 (50-100K) | 拆分为2-3个子任务 |
| 超大任务 (>100K) | 多轮对话，每轮独立 |

## 五、测试框架

### 5.1 测试工具
| 工具 | 用途 | 说明 |
|------|------|------|
| pytest | 单元测试 | Python标准测试框架 |
| Playwright | 端到端测试 | Web UI测试 |
| coverage | 覆盖率 | 代码覆盖率分析 |

### 5.2 安装（通过Cline执行）
```python
# 提示词示例
"""
在项目中:
1. 安装pytest: pip install pytest
2. 安装playwright: pip install playwright  
3. 运行测试: pytest tests/ -v
"""
```

## 六、重要规则

### 6.1 禁止事项
- ❌ **禁止将外部AI模型加入我的配置**
- ❌ **不依赖单一模型完成所有任务**
- ❌ **不在对话中保留完整大型代码**

### 6.2 必须事项
- ✅ **所有代码生成通过外部助手执行**
- ✅ **每次代码变更必须测试验证**
- ✅ **复杂任务必须拆分子任务**
- ✅ **保留完整记忆供后续参考**

## 八、适用场景

此工作模式适用于：
1. **大型项目开发** - 需要多个模块协同
2. **复杂功能实现** - 超出单次上下文限制
3. **测试驱动开发** - 需要完整测试覆盖
4. **多语言支持** - 调用不同技术栈的AI助手
5. **长期项目维护** - 记忆化管理任务状态
