# schema_generator.py

> 用户路径扫描引擎，生成 `user_schema.yaml`
> V4.2 Auto-Extract 核心组件

---

## 功能

1. **扫描 `~/.openclaw/workspace/`**（递归），收集所有 `.md` `.yaml` `.yml` `.json` `.txt` `.sql` `.py` 文件
2. **扫描三个 HIPPO 版本目录**：
   - `/mnt/x/CLABIN/HIPPO/hippocampus-for-github/`
   - `/mnt/x/CLABIN/HIPPO/hippocampus-for-clawhub/`
   - `/mnt/x/CLABIN/HIPPO/hippocampus-for-usb/`
   -（自动排除 `assets/` 运行时生成目录）
3. **读取 `base_schema.yaml`**（`../task_schema/base_schema.yaml`）作为参考骨架
4. **输出 `user_schema.yaml`**，包含用户实际文件路径清单

---

## 输出文件

| 文件 | 路径 |
|------|------|
| `user_schema.yaml` | `scripts/auto_extract/user_schema.yaml` |
| `base_schema.yaml` | `../task_schema/base_schema.yaml`（参考用，可不存在） |

---

## 使用方法

```bash
# 标准运行
python schema_generator.py

# 静默模式（只输出文件，不打印详情）
python schema_generator.py -q
```

```python
# 作为模块调用
from schema_generator import generate_user_schema
schema = generate_user_schema()
print(schema["total_files"])
```

## 在 V4.2 工作流中的位置

```
base_schema.yaml          ← 官方骨架（已知路径）
     ↓
schema_generator.py       ← 扫描用户实际路径 → user_schema.yaml
     ↓
extract_runner.py         ← 对比两者差异，备份新文件
```
