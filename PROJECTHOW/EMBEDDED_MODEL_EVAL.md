# 内嵌小型 LLM 模型可行性评估

**项目**：HIPPO V4 USB 分发版 LLM 评估  
**作者**：ran（ResearchAnalyst）  
**日期**：2026-04-22  
**状态**：初稿

---

## 一、评估背景与目标

HIPPO V4 USB 版需要评估是否能在不离网的情况下，为用户提供本地 AI 对话能力。本报告评估适合 USB 分发的轻量级 LLM 模型，重点关注：
- 体积（必须 ≤ 4 GB）
- 中文能力（核心需求）
- CPU 推理可行性（无需 GPU）
- 与 USB 分发架构的兼容性

> 体积数据基于公开规格，INT4 量化使用 llama.cpp 标准量化参数，未实测。

### 2.2 中文能力详解

**Qwen2 系列（阿里巴巴）**：
- 专为中文优化，训练数据中中文占比高
- 0.5B 和 1.5B 版本在中文对话上表现优秀
- 思维链（CoT）和工具调用能力在同规模中领先
- 开源协议：Apache 2.0，商业友好

**Phi-3-mini（微软）**：
- 英文为主，中文能力尚可但不如 Qwen2
- 逻辑推理能力强，适合代码和复杂问题
- 微调版本的中文表现差异大，需选对模型版本
- 开源协议：MIT

**DeepSeek-Lite**：
- 国产模型，中文优化较好
- 体积小，适合嵌入式场景
- 开源协议：DeepSeek License

**TinyLlama**：
- 英文为主，中文能力弱
- 仅适合英文为主的极轻量场景

## 三、架构评估：USB 代码 + 本地模型 + 用户 PC

### 3.1 推荐架构

```
┌─────────────────────────────────────────────────────┐
│                    USB 设备                          │
│  ┌─────────────┐  ┌──────────────┐  ┌────────────┐ │
│  │ HIPPO 代码   │  │ ChromaDB 数据 │  │ Qwen2-0.5B │ │
│  │ (~65 MB)   │  │ (~50 MB)     │  │ (~700 MB)  │ │
│  │             │  │              │  │ (INT4量化)  │ │
│  └─────────────┘  └──────────────┘  └────────────┘ │
└────────────────────────┬────────────────────────────┘
                         │ USB 连接
                         ▼
┌─────────────────────────────────────────────────────┐
│                   用户 HOST PC                        │
│  ┌─────────────┐  ┌──────────────┐  ┌────────────┐ │
│  │ Python 环境  │  │ llama.cpp    │  │  CPU 推理   │ │
│  │ (已有或安装)  │  │  运行时      │  │  内存 ~2GB  │ │
│  └─────────────┘  └──────────────┘  └────────────┘ │
│                                                     │
│  ← LLM 模型文件由 USB 加载到 PC 内存运行 →           │
└─────────────────────────────────────────────────────┘
```

### 3.2 数据流说明

1. **模型文件**：存储在 USB 上，由 PC 的 CPU 加载到内存运行（不是嵌入式 CPU 执行）
2. **ChromaDB 数据**：存储在 USB，索引查询在 PC 内存进行
3. **Schema 数据**：YAML 存储在 USB，应用层读取

**关键约束**：USB 只是存储介质，所有计算在 HOST PC 进行。

### 3.3 模型放置策略

| 策略 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| **策略A：模型全量放在 USB** | 模型文件存储在 USB | 用户插入即可用，无需联网 | 模型大（~700 MB），USB 读写较慢 |
| **策略B：首次运行下载** | 启动时从网络下载模型到 PC 本地 | USB 体积小，首发包更小 | 首次需要网络，无法完全离线 |
| **策略C：分阶段下载** | 提供多个模型版本供选择下载 | 灵活，用户按需选择 | 增加复杂度 |
| **策略D：放在 PC 本地** | 模型存储在 PC 本地目录（非 USB） | USB 读写不影响推理速度 | 多台电脑需要分别配置 |

**推荐**：策略A（全量在 USB）+ 策略D（PC 本地）双轨支持：
- 优先使用 PC 本地模型（如用户已有 ollama）
- USB 模型作为备用/离线保障

## 四、推理引擎评估（llama.cpp / ollama）

### 4.1 llama.cpp

**优势**：
- 纯 C/C++ 实现，跨平台
- 支持 CPU 推理（无 GPU 也可运行）
- INT4/INT8 量化效果好
- 与 Python 集成成熟（llama-cpp-python 包）
- 被几乎所有本地 LLM 工具链使用

**性能（Qwen2-0.5B INT4，CPU 推理）**：

| CPU 类型 | 内存 | 推理速度 | 说明 |
|---------|------|---------|------|
| Intel i5/i7（近3年）| 16 GB RAM | ~15-25 tok/s | 流畅对话 |
| Intel i3 / AMD 锐龙3 | 8 GB RAM | ~8-15 tok/s | 可接受 |
| 老旧 CPU（5年以上）| 8 GB RAM | ~3-8 tok/s | 较慢，但可用 |

> 未实测，数据参考 llama.cpp 社区报告。

**安装依赖**：
```bash
pip install llama-cpp-python
# 约 50-100 MB
```

### 4.2 ollama

**优势**：
- 开箱即用的模型管理和推理工具
- 一条命令运行模型：`ollama run qwen2:0.5b`
- 模型自动下载，自动管理
- 跨平台支持

**USB 分发时的限制**：
- ollama 模型默认存储在 `~/.ollama/models/`
- 不方便将模型"绑定"到 USB 一起带走
- 适合作为 PC 预装的推理工具，而非 USB 产品的一部分

**推荐用法**：
- USB 产品文档建议用户"如需本地 LLM，请先在 PC 安装 ollama"
- HIPPO 检测到 ollama 时自动调用，否则提示安装

## 五、模型推荐与 USB 分发策略

### 5.1 模型推荐优先级

| 优先级 | 模型 | 理由 | 适用场景 |
|--------|------|------|---------|
| ⭐⭐⭐⭐⭐ 第一推荐 | **Qwen2-0.5B** | 中文优秀、体积小、CPU 可跑、Apache 2.0 | USB 全功能版首选 |
| ⭐⭐⭐⭐ 第二推荐 | **Qwen2-1.5B** | 中文更优，体积略大（~1.5GB） | PC 内存 16GB+ 用户推荐 |
| ⭐⭐⭐ 第三推荐 | **Phi-3-mini** | 英文/逻辑强，中文尚可 | 混合语言场景 |
| ⭐⭐ 备选 | **DeepSeek-Lite-1.8B** | 国产、中文良好、体积小 | 作为 Qwen2 备选 |

### 5.2 量化格式推荐

**推荐 INT4 量化**（GGUF 格式）：
- 体积最小（Qwen2-0.5B 可压至 ~600 MB）
- 质量损失可接受（对对话场景影响小）
- 被 llama.cpp 原生支持

**不推荐**：
- FP16：体积过大，USB 分发不友好
- INT2 及以下：质量损失过大，实用性差

### 5.3 USB 分发模型文件格式

```
HIPPO_USB/
└── models/                  # LLM 模型目录
    ├── qwen2-0.5b-instruct-q4_0.gguf  # Qwen2 INT4 量化（约 600 MB）
    └── config.json          # 模型配置（约 2 KB）
```

**关于首次运行下载**：
- 当前 16 GB USB 方案中，模型（~700 MB）占主要体积
- 如果要进一步压缩首发包体积，可选择"首次运行下载"策略
- 但 HIPPO 作为"离线 AI 产品"，应尽量减少联网依赖

## 七、风险与限制

### 7.1 主要风险

| 风险 | 说明 | 缓解措施 |
|------|------|---------|
| CPU 推理速度慢 | 老旧 PC 可能仅 3-5 tok/s | 提供速度模式选项，降低模型质量换速度 |
| 中文分词效果差 | 部分小模型中文训练不足 | 优先选择 Qwen2（阿里巴巴中文优化） |
| 模型法律合规 | 部分模型需遵守许可协议 | Qwen2 采用 Apache 2.0，完全友好 |
| USB 读写速度 | USB 2.0 速度慢，模型加载慢 | 建议 USB 3.0+；模型加载到 PC 本地后不再读 USB |
| 内存不足 | 8 GB 以下 PC 可能不够 | 添加内存检测，内存不足时关闭 LLM 功能 |

### 7.2 不适合 LLM 的用户

- RAM < 6 GB 的 PC（无法流畅运行）
- 老旧 CPU（Intel Haswell 之前，约 2013 年前）
- USB 2.0（模型加载速度极慢）
- 需要实时流式输出的用户体验差

## 八、最终建议

### 8.1 模型选型

| 分发版本 | 模型 | 理由 |
|---------|------|------|
| **标准版（16GB USB）** | Qwen2-0.5B INT4 | 体积与质量平衡最佳，中文优秀 |
| **PC 预装版** | Qwen2-1.5B INT4 | PC 内存 16GB+ 时使用更大模型 |

### 8.2 USB 分发内容清单

```
HIPPO_USB_16GB/
├── brain_dump/           # 主代码（~65 MB）
├── models/               # LLM 模型
│   └── qwen2-0.5b-instruct-q4_0.gguf  # ~600 MB
├── chroma_data/          # ChromaDB 数据（可选，约 50 MB）
├── schema/               # Schema YAML 数据
└── ...                   # 其他资源
```

### 8.3 关键结论

1. **Qwen2-0.5B 是 USB 分发最佳选择**：中文优秀、体积小（INT4 仅 ~600 MB）、CPU 可跑、协议友好
2. **最低 PC 配置**：8 GB RAM，Python 3.8+，USB 3.0
3. **推理速度**：现代 PC（约 15-25 tok/s）可流畅对话，老旧 PC（约 5-10 tok/s）勉强可用
4. **架构清晰**：USB 存储 + PC 计算分离，模型文件无需在嵌入式设备运行
5. **建议首发包优先 Scenario B**（16GB USB），LLM 作为可选组件或后续版本发布
