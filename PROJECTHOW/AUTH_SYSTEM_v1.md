## 一、设计哲学

| 原则 | 说明 |
|------|------|
| **零门槛注册** | 无手机号、无邮箱（邮箱选填，仅用于推广） |
| **令牌即身份** | 用户自选令牌 = 用户名 + 密钥种子 |
| **快速校验** | 每次操作（读/写/整理）做本地密钥核对，<100ms |
| **适度防破解** | 不追求军事级，让破解成本 > 购买成本即可 |
| **分级分发** | 下载版（自动植入密钥）+ U盘版（物理植入密钥） |

## 二、角色定义

### 角色1：用户（User）
- 注册：自设令牌（8-20位，大小写字母+数字）
- 权限：读取、写入、整理、导出自己的记忆数据
- 不涉及：后台管理、其他用户数据

### 角色2：开发者/商家（Admin）= 鱼总
- 拥有：发布密钥生成器、用户管理、分发自动化
- 工具：密钥生成脚本 + 分发打包脚本 + 到期管理
- 不涉及：读取用户记忆数据（纯本地产品，无服务端）

## 三、令牌与密钥体系

### 3.1 令牌（Token）

```
定义：用户自行设定的身份标识
格式：8-20位，至少含1个大写、1个小写、1个数字
性质：公开的（类似用户名），但同时也是密钥推导的种子
示例：FishMaster2026
```

### 3.2 激活密钥（Activation Key）生成流程

```
用户令牌: "FishMaster2026"  (用户自设)
官方认证字符: "HP2026"       (鱼总持有，每批次可不同)
到期日: "2027-05-21"         (付费周期)

Step 1: 拼接原始串
  raw = Token + "|" + AuthChar + "|" + ExpiryDate
      = "FishMaster2026|HP2026|2027-05-21"

Step 2: HMAC-SHA256 签名
  secret = 鱼总持有的主密钥（32字节，离线保存）
  signature = HMAC-SHA256(secret, raw)

Step 3: 取前16字节 → Base32编码 → 格式化
  activation_key = Base32(signature[:16])
                 = "JBSWY3DPEHPK3PXP"  (示例)

Step 4: 组装完整激活码
  full_key = Token[:4].upper() + "-" + activation_key + "-" + ExpiryDate[-4:]
           = "FISH-JBSWY3DPEHPK3PXP-0521"
```

### 3.3 密钥文件格式

用户首次激活后，在本地生成 `.hippo_license`：

```json
{
  "version": 1,
  "token_hash": "sha256_of_token_lowercase",  // 不存原文，存哈希
  "activation_key": "FISH-JBSWY3DPEHPK3PXP-0521",
  "expires_at": "2027-05-21",
  "machine_id": "hashed_hwid",                 // 绑定机器（可选）
  "features": ["read", "write", "organize"]
}
```

### 3.4 校验流程（每次操作触发）

```
用户打开 Hippocampus:
  ① 读取 .hippo_license
  ② 用本地存的 token_hash 验证令牌一致性
  ③ 用内置的公钥验证 activation_key 签名
  ④ 检查 expires_at 是否过期
  ⑤ 检查 machine_id 是否匹配（可选）
  全部通过 → 显示 "已激活 · 到期 YYYY-MM-DD"
  任一失败 → 进入受限模式（只读或禁止使用）
```

## 四、安全层级设计

### 防护目标："破解成本 > 购买成本"

| 攻击路径 | 防护措施 | 破解难度 |
|----------|---------|---------|
| 直接复制 `.hippo_license` | machine_id绑定（可选开启） | 中 |
| 逆向推导主密钥 | HMAC-SHA256，不存主密钥在客户端 | 高 |
| 修改过期时间 | 签名覆盖到期日，改了签名就废 | 高 |
| 找到通用激活码 | 每个用户令牌不同，签名不同 | 高 |
| 直接patch跳过校验 | 代码混淆 + 多点校验 | 中 |
| 共享一个激活码 | machine_id绑定（可选） | 中 |

### 关于machine_id绑定的权衡

| 模式 | 优点 | 缺点 | 适用 |
|------|------|------|------|
| **不绑定**（推荐起步） | 换电脑无缝，用户友好 | 一个码多人用 | 低价策略 |
| **软绑定**（宽松） | 允许2-3台设备 | 需要计数机制 | 中等价格 |
| **硬绑定** | 一机一码 | 换电脑需重新激活 | 高价产品 |

**建议v1.0先用"不绑定"，后续根据盗版情况升级为"软绑定"。**

## 五、分发流程

### 5.1 下载版（网站）

```
用户在网站:
  ① 输入令牌 → 生成唯一下载包
  ② 付费成功 → 后端生成 activation_key
  ③ activation_key 自动注入到下载包的 license模板
  ④ 提供专属下载链接（含令牌hash作为文件名的一部分）
  
  技术实现：动态打包脚本（Python）
    - 读取用户令牌 + 到期日
    - 调用密钥生成器
    - 将密钥写入 hippocampus/license/.hippo_license
    - zip打包 → 提供下载
```

### 5.2 U盘版（物理发货）

```
鱼总端:
  ① 用户提交令牌 + 付费
  ② 鱼总运行密钥生成脚本:
     python3 keygen.py --token "FishMaster2026" --days 365
     → 输出: FISH-JBSWY3DPEHPK3PXP-0521
  ③ 将密钥写入U盘:
     - U盘已有 hippocampus/ 完整程序
     - 将 .hippo_license 写入 hippocampus/license/ 目录
  ④ 发货
  
  自动化脚本（鱼总用）:
     python3 burn_usb.py --token "xxx" --device /dev/sdX --days 365
     → 自动: 格式化 + 复制程序 + 注入密钥 + 验证
```

### 5.3 密钥生成器（鱼总专用，离线工具）

```
文件: /mnt/x/CLABIN/HIPPO/tools/keygen.py

功能:
  - 输入: 用户令牌 + 天数
  - 输出: 完整激活码
  - 依赖: 主密钥文件（master.key，离线保存，绝不分发）
  - 安全: master.key 只存在于鱼总的开发机上

文件: /mnt/x/CLABIN/HIPPO/tools/burn_usb.py

功能:
  - 输入: U盘设备路径 + 用户令牌 + 天数
  - 流程: 调用keygen → 写入U盘 → 验证
```

## 七、用户体验流程

### 首次使用（已植入密钥的U盘/下载包）
```
双击autorun → GUI启动
→ 自动读取 .hippo_license
→ 校验通过
→ 显示 "✅ FishMaster2026 · 有效期至 2027-05-21"
→ 正常使用
```

### 首次使用（未激活/试用版）
```
双击autorun → GUI启动
→ 未找到 .hippo_license
→ 显示激活向导:
   "请输入您的令牌（注册时设置的用户名）"
   "请输入激活码"
   [激活] 按钮
→ 校验通过 → 写入 .hippo_license
→ 正常使用
```

### 到期提醒
```
距到期 < 30天: 
  状态栏黄色提示 "⚠️ 授权将于 XX 天后到期，请续费"
  功能不受限

已到期:
  状态栏红色 "🔴 授权已过期"
  进入受限模式（只读，不能写入新记忆）
  提示续费链接
```
