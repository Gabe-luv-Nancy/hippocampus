## 1. 概述

### 1.1 产品背景

Hippocampus Client 是本地桌面客户端，用户购买软件后通过该客户端：
1. 配置 AI 工具监控规则（本地路径）
2. 配置 SSH 云端备份路径
3. 插入 HIPPOCAMPUS U 盘触发自动备份

客户端默认最小化到系统托盘，后台常驻运行。

### 1.2 目标用户

- 有多个 AI 写作工具（OpenClaw、Kimi、ChatGLM 等）的重度用户
- 需要本地 + 云端双备份的数据安全导向用户
- 非技术背景，但能接受"配置路径"的进阶用户

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| **零学习成本** | 界面与常见网盘客户端（Dropbox/OneDrive）一致 |
| **所见即所得** | 配置项与实际行为一一对应，无隐藏逻辑 |
| **容错友好** | 错误状态明确，恢复路径清晰 |
| **跨平台一致** | Windows / macOS / Linux 三端 UI 统一 |

|------|------|

| **[查看日志]** | 打开"界面4：日志对照"弹窗 |
| **状态面板** | 实时刷新，U盘断开时"已连接"变"⚠️ 未连接" |

#### 状态说明

| 状态 | 视觉表现 |
|------|---------|
| U盘已连接 | 🟢 绿色 + 文字"已连接" |
| U盘未连接 | 🔴 红色 + 文字"未连接" |
| 备份中 | 🟡 黄色 + 旋转图标 + 文字"备份中..." |
| 备份失败 | 🔴 红色 + 文字"失败" + [重试]按钮 |

#### ASCII 原型

```
┌─────────────────────────────────────────────────────────────┐
│  配置 OpenClaw                                      [×]   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  监控路径：                                                 │
│  ┌─────────────────────────────────────────────────────┐ │
│  │ /home/user/.openclaw/workspace/          [删除]      │ │
│  │ /home/user/.openclaw/memory/              [删除]      │ │
│  │ [+ 添加路径]                                           │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                             │
│  文件过滤规则：                                             │
│  ┌─────────────────────────────────────────────────────┐ │
│  │ 包含模式：  *.md, *.json, *.yaml, *.txt             │ │
│  │ 排除模式：  **/__pycache__/**, **/node_modules/**   │ │
│  │ 文件大小：  最小 0 KB  最大 50 MB                    │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                             │
│  [保存]  [取消]                                            │
└─────────────────────────────────────────────────────────────┘
```

#### 交互逻辑

| 元素 | 行为 |
|------|------|
| **[×] 关闭** | 关闭弹窗，放弃未保存的更改 |
| **[删除]（路径行）** | 删除该路径，路径列表最少保留1个 |
| **[+ 添加路径]** | 弹出系统文件浏览器选择目录 |
| **包含模式** | 默认：`*.md, *.json, *.yaml, *.txt`，可自定义 |
| **排除模式** | 默认：`**/__pycache__/**, **/node_modules/**`，支持 glob 语法 |
| **文件大小限制** | 超出限制的文件不备份，不占空间 |
| **[保存]** | 保存配置，关闭弹窗，主界面实时刷新 |
| **[取消]** | 放弃更改，关闭弹窗 |

#### 校验规则

- 监控路径至少保留 1 个（不可全删）
- 排除模式与包含模式不能完全冲突（全排除了就没东西可备份）
- 文件大小上限最大 500MB（防止误选大文件）

```
├─────────────────────────────────────────────────────────────┤
│                                                             │

│                                                             │
│  连接信息：                                                 │
│  地址：     [123.45.67.89______________]  端口：[22]      │
│  用户名：   [ubuntu____________________]                    │
│  认证方式： ○密码  ●密钥文件                                │
│  密钥文件： [~/.ssh/id_rsa___________] [浏览...]           │
│                                                             │
│  监控路径：                                                 │
│  ┌─────────────────────────────────────────────────────┐ │
│  │ ~/.openclaw/workspace/                    [删除]      │ │
│  │ /data/ai_records/                        [删除]      │ │
│  │ [+ 添加路径]                                           │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                             │
│  [测试连接]  [保存]  [取消]                                │
└─────────────────────────────────────────────────────────────┘
```

#### 交互逻辑

| 元素 | 行为 |
|------|------|
| **主机名称** | 纯文字标识，用于列表显示，必填 |
| **地址 + 端口** | IPv4/IPv6/域名均支持，默认端口22 |
| **认证方式 - 密码** | 首次连接弹出密码输入框，密码不持久化（仅本次使用） |
| **认证方式 - 密钥文件** | 点击[浏览...]选择私钥文件，支持 `~/.ssh/id_rsa`、`id_ed25519` 等 |
| **[测试连接]** | 后台执行 `ssh -o ConnectTimeout=10`，5秒内响应成功/失败 |
| **监控路径** | 同界面2，支持多路径 |
| **[保存]** | 通过连接测试后保存，失败弹窗提示 |

#### 错误提示

| 错误场景 | 提示文字 |
|---------|---------|
| 连接超时 | ❌ 连接超时（10秒），请检查地址和网络 |
| 认证失败 | ❌ 认证失败，请检查用户名和密钥文件 |
| 路径不存在 | ❌ 路径 `~/.openclaw/workspace` 在远程不存在 |
| 保存未测试 | ⚠️ 请先点击"测试连接"保存 SSH 主机 |

### 界面 4：日志对照（弹窗）

#### ASCII 原型

```
┌─────────────────────────────────────────────────────────────┐
│  日志对照：客户端 vs U盘                            [×]   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  过滤：[全部▼] [FOUND▼] [SKIP▼] [RECEIVED▼] [ERROR▼]       │
│                                                             │
│  客户端日志                        U盘日志                   │
│  ────────────────────────────── ──────────────────────── │
│  [10:01:02] FOUND OpenClaw      [10:01:04] RECEIVED OK  │
│  MEMORY.md (3.2KB) sha256:abc123  MEMORY.md ✓            │
│                                                             │
│  [10:01:02] FOUND OpenClaw      [10:01:04] RECEIVED OK   │
│  TODO.md (1.1KB) sha256:def456  TODO.md ✓                │
│                                                             │
│  [10:01:03] SKIP OpenClaw       (无对应记录-正常关闭)     │
│  AGENTS.md hash_match           U盘本次未收到此文件       │
│                                                             │
│  ───────────────────────────────────────────────────────── │
│  统计：FOUND 15 / SKIP 12 / RECEIVED 3 / ERROR 0           │
└─────────────────────────────────────────────────────────────┘
```

#### 交互逻辑

| 元素 | 行为 |
|------|------|
| **[×]** | 关闭弹窗 |
| **过滤器下拉菜单** | 单选：全部 / FOUND / SKIP / RECEIVED / ERROR，实时筛选 |
| **时间戳** | 格式 `HH:MM:SS`，精确到秒 |
| **SHA256** | 每文件独立 hash，用于比对是否真正相同 |
| **双行对照** | 客户端一行 + U盘一行，左侧客户端为主动侧 |
| **统计栏** | 底部显示本次会话各类事件计数 |

#### 日志类型说明

| 类型 | 含义 | 颜色 |
|------|------|------|
| FOUND | 客户端发现新文件或修改过的文件 | 蓝 |
| SKIP | 文件 hash 与记录一致，跳过 | 灰 |
| RECEIVED | U盘确认接收文件 | 绿 |
| ERROR | 传输/校验错误 | 红 |

## 4. 技术实现方案

### 4.1 推荐技术栈

| 方案 | 框架 | 适用平台 | 优点 | 缺点 |
|------|------|---------|------|------|
| **首选** | **Tauri v2** | Win/Mac/Linux | 体积小（<10MB）、Rust后端安全、原生体验 | 前端需学 Rust 小众生态 |
| **备选** | **Electron** | Win/Mac/Linux | 生态成熟、Web技术栈、调试方便 | 体积大（>100MB）、内存占用高 |
| **保守** | **PyQt6** | Win/Mac/Linux | Python原生、GUI成熟 | 样式定制成本高、Linux体验差异大 |
| **轻量优先** | **Tkinter** | Win/Mac/Linux | 内置Python、无需安装额外包 | 样式老旧、不适合复杂UI |

### 4.2 推荐：Tauri + React/Vue

**架构：**

```
┌─────────────────────────────────────────────────────┐
│                    Tauri 架构                       │
├──────────────┬──────────────────┬───────────────────┤
│   Frontend   │   Rust Backend   │   System Layer    │
│   (React/Vue)│   (Tauri Core)    │                   │
│              │                  │                   │
│  - React 18  │  - 文件监控       │  - 系统托盘       │
│  - Tailwind  │  - SSH/SFTP      │  - 文件系统访问   │
│  - i18n      │  - 备份逻辑       │  - USB 检测       │
│              │  - 哈希校验       │  - 原生通知       │
└──────────────┴──────────────────┴───────────────────┘
```

**理由：**
- 体积比 Electron 小 10 倍，适合 U 盘分发
- Rust 后端在文件监控和 SSH 场景性能优秀
- 支持系统托盘原生实现，跨平台一致性好

### 4.3 核心模块设计

```
src/
├── main.rs              # Tauri 入口，托盘初始化
├── commands/
│   ├── mod.rs
│   ├── fs_monitor.rs    # 文件系统监控（notify crate）
│   ├── ssh_sync.rs      # SSH/SFTP 同步
│   ├── usb_detect.rs    # USB 插拔检测
│   └── backup.rs        # 备份核心逻辑
├── models/
│   ├── config.rs        # 配置结构体
│   ├── ai_tool.rs       # AI工具监控配置
│   └── ssh_host.rs      # SSH主机配置
└── db/
    └── sqlite.rs        # 本地配置存储（SQLite）
```

### 4.4 跨平台兼容说明

| 功能 | Windows | macOS | Linux |
|------|---------|-------|-------|
| 系统托盘 | ✅ 支持 | ✅ 支持 | ✅ 支持 |
| USB 检测 | ✅ WMI | ✅ IOKit | ✅ libusb |
| 文件监控 | ✅ inotify/wmi | ✅ FSEvents | ✅ inotify |
| SSH/SFTP | ✅ 通用 | ✅ 通用 | ✅ 通用 |
| 原生通知 | ✅ Windows Toast | ✅ Apple Notification Center | ✅ libnotify |

## 5. 数据流设计

```
[本地文件变化]
    │
    ▼
[文件监控模块（notify）] ──检测变化──▶ [变更队列]
    │                                    │
    ▼                                    ▼
[哈希计算（SHA256）]           [备份任务生成]
    │                                    │
    ▼                                    ▼
[查询本地记录（SQLite）] ──比对hash──▶ [决策：备份/跳过]
    │                                    │
    │                              ┌────┴────┐
    ▼                              ▼         ▼
[写入备份记录]              [U盘写入]  [SSH云端同步]
```

## 6. 配置文件格式

客户端配置存储于 `~/.hippocampus/config.json`：

```json
{
  "version": "3.1.0",
  "ai_tools": [
    {
      "id": "openclaw",
      "name": "OpenClaw",
      "enabled": true,
      "paths": [
        "/home/user/.openclaw/workspace/",
        "/home/user/.openclaw/memory/"
      ],
      "include_patterns": ["*.md", "*.json", "*.yaml", "*.txt"],
      "exclude_patterns": ["**/__pycache__/**", "**/node_modules/**"],
      "max_file_size_kb": 51200
    }
  ],
  "ssh_hosts": [
    {
      "id": "host-001",
      "name": "云服务器-A",
      "host": "123.45.67.89",
      "port": 22,
      "user": "ubuntu",
      "auth_type": "key",
      "key_path": "~/.ssh/id_rsa",
      "paths": [
        "~/.openclaw/workspace/",
        "/data/ai_records/"
      ]
    }
  ],
  "backup": {
    "auto_backup_on_usb": true,
    "last_backup_date": "2026-04-20",
    "last_sync_time": "10:01:00"
  }
}
```
