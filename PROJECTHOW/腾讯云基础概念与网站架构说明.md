## 一、腾讯云核心概念

### 1.1 云服务器（VPS / CVM）

**全称：** Cloud Virtual Machine / 云服务器 CVM

**通俗理解：** 你在腾讯云上租的一台电脑（Linux 系统），它有固定的 IP 地址（81.70.200.211），24 小时开着，任何时候都可以连接使用。

**特点：**
- 拥有完整的 Linux 文件系统
- 可以安装任何软件（nginx、Python、数据库等）
- 按月/年付费，属于你自己的计算资源
- **和你的本地电脑一样，文件存在本地磁盘上**

**腾讯云轻量应用服务器（Lighthouse）：** 腾讯云推出的简化版 CVM，套餐式收费，更便宜，适合个人用户。

---

### 1.2 腾讯云对象存储（COS）

**全称：** Cloud Object Storage

**通俗理解：** 腾讯云提供的"网盘"服务。专门用来存文件（图片、视频、文档等），通过 API 或网页上传下载，不是一台完整的电脑，没有操作系统。

**特点：**
- 存储空间几乎是无限的
- 按存储量和流量收费
- 通过 URL 直接访问文件（适合放图片、视频等静态资源）
- **不占用云服务器的计算资源**

**典型用途：**
- 存网站图片/视频（CDN 加速）
- 存大文件，用 COS 的下载功能而不是走服务器流量
- 备份文件

---

### 1.3 两者关系对比

| 对比项 | 云服务器 CVM | 对象存储 COS |
|--------|-------------|-------------|
| 本质 | 一台电脑 | 一个网盘 |
| 操作系统 | 有 Linux | 没有 |
| 适用场景 | 运行程序、搭网站 | 存文件 |
| 费用 | 固定月费 | 按用量计费 |
| 访问方式 | IP / 域名 / SSH | API / URL |
| 和网站关系 | 网站程序跑在这里 | 网站资源可以放这里 |

## 二、腾讯云控制台的两个入口

### 2.1 Web 管理控制台

**地址：** [cloud.tencent.com](https://cloud.tencent.com)

**登录方式：** 微信扫码 / 手机扫码 / 账号密码

**是什么：** 腾讯云提供的网页版"遥控器"。你在网页上点任何按钮（创建服务器、配置防火墙），本质都是调用腾讯云的 API 来操控你的资源。

**能做什么：**
- 创建/删除云服务器实例
- 配置安全组（防火墙）
- 购买域名、管理 DNS 解析
- 开通 COS、数据库等各种服务
- 查看账单

---

### 2.2 命令行界面（云服务器登录）

**是什么：** 直接连接到你云服务器（81.70.200.211）的 Linux 系统。

**两种打开方式：**

| 方式 | 怎么进 | 实际是什么 |
|------|--------|-----------|
| 控制台 → 实例 → "登录" 按钮 | 网页上点 | VNC（网页版远程桌面，像看黑屏） |
| 本地 Terminal SSH | `ssh root@81.70.200.211` | 通过 SSH 协议远程连接 |

**两者的区别：**
- VNC 不需要网络开放 22 端口，适合临时应急
- SSH 更稳定、功能更强，是日常连接云服务器的标准方式

**连接命令（需要 SSH 密钥）：**
```bash
ssh -i ~/.ssh/id_ed25519 root@81.70.200.211
```

## 三、网站是怎么跑起来的

### 3.1 完整访问流程

```
用户浏览器输入域名或IP
        ↓ DNS 解析
找到你的云服务器 IP（81.70.200.211）
        ↓
云服务器收到请求（端口 80/443）
        ↓
Nginx（Web 服务器软件）接收请求
        ↓
Nginx 从 /var/www/html/ 目录里找对应文件
        ↓
返回 HTML/CSS/图片 等给用户浏览器
```

### 3.2 核心目录结构

```
/var/www/html/          ← 网站根目录（Nginx 在这里找文件）
├── index.html          ← 首页
├── about/
│   └── index.html     ← 关于页
├── downloads/
│   ├── index.html     ← 下载列表页
│   └── 文件.zip        ← 实际可下载的文件
└── assets/             ← 静态资源（CSS/JS/图片）

/root/clabin_sync/      ← CLABIN 同步目录（和网站无关）
├── ESTUDIAR/          ← 你的视频学习项目
├── LOTT/              ← 量化交易项目
└── HIPPO/             ← 本文档所在位置
```

### 3.3 Nginx 是什么

**Nginx：** Web 服务器软件，负责接收用户浏览器的请求，找到对应文件，返回给浏览器。

**类比：** Nginx 就像餐厅服务员——用户"点餐"（发请求），Nginx 去"厨房"（/var/www/html/）拿"菜品"（网页文件），端给用户。

**配置文件位置：** `/etc/nginx/conf.d/clabin.conf`

## 四、CLABIN 目录和网站的关系

### 4.1 两者完全独立

```
/root/clabin_sync/      ← Syncthing 同步目录（你本地也有）
     │
     └── 和 /var/www/html/ 完全不搭嘎

/var/www/html/          ← 网站文件目录（只有云端有）
     │
     └── Nginx 读取这里，服务给用户
```

**CLABIN 里的内容（ESTUDIAR、LOTT、HIPPO 等）不会出现在网站上。**
用户访问你的 IP，看到的只是 `/var/www/html/` 里的文件。

### 4.2 如果把网站文件放到 CLABIN 里？

**技术上可行，但需要修改 Nginx 配置。**

**问题：**
- Nginx 当前配置是 `root /var/www/html/`
- 如果把网站文件搬到 `/root/clabin_sync/website/`，Nginx 就找不到了

**解决方案（如果需要）：**
1. 修改 Nginx 配置指向新目录：`root /root/clabin_sync/website/`
2. 或者用软链接：`ln -s /root/clabin_sync/website /var/www/html`
3. 重载 Nginx：`nginx -s reload`

**但实际上没必要这么做——**
- `/var/www/html/` 是 Linux 标准网站目录，设计出来就是放网站文件的
- CLABIN 是 Syncthing 同步用的，两者各司其职最好

## 五、SSL 证书与 HTTPS

### 5.1 HTTP vs HTTPS

| 协议 | 说明 |
|------|------|
| HTTP | 明文传输，数据不加密（地址栏显示"不安全"） |
| HTTPS | 加密传输（需要 SSL 证书） |

### 5.2 SSL 证书

**是什么：** 相当于网站的"身份证"，证明这个域名属于你。

**免费方案：** Let's Encrypt（自动申请、自动续期）

**申请后：**
- 网站支持 HTTPS 访问
- 浏览器地址栏显示小锁 🔒
- 微信/支付宝等支付接口必须 HTTPS 才能接入
