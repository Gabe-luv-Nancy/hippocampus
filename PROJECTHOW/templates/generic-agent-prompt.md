# HIPPO 记忆注入引导 — 通用 Agent 版

> **使用方式：** 将下方「👇 从这里开始复制 👇」到「👆 到这里结束复制 👇」之间的全部内容，**复制粘贴**到你与 Agent 的对话中发送即可。Agent 会在此后的对话中自动执行记忆写入流程。

---

## 👇 从这里开始复制 👇

```
【HIPPO 记忆系统 — 对话指令】

你好，从现在开始，请在每次对话轮次结束时执行以下「记忆写入」流程：

---

### 一、你的职责

1. **每次回复我之后**，自动在回复末尾附上一段「📝 记忆摘要」区块。
2. 该区块的内容格式严格遵循下方定义。
3. 如果我明确说「不需要记忆」或「跳过记忆」，则本次不生成。

---

### 二、记忆摘要格式

每次对话结束时，请按以下格式输出：

```
---
## 📝 记忆摘要

**日期：** YYYY-MM-DD
**主题：** 本次对话的核心主题（一句话）
**标签：** #标签1 #标签2 #标签3

### 关键信息
- 要点1：...
- 要点2：...
- 要点3：...

### 待办 / 后续
- [ ] （如有待办事项列出，没有则写「无」）

### 记忆文件建议路径
`~/.hippo_memory/YYYY/YYYY-MM-DD.md`
```

---

### 三、记忆文件存储说明

记忆文件的**标准目录**为：

```
~/.hippo_memory/
├── 2026/
│   ├── 2026-01-15.md
│   ├── 2026-01-16.md
│   └── ...
└── index.md          ← 总索引文件
```

- 每日一个文件，文件名为 `YYYY-MM-DD.md`
- `index.md` 中按时间倒序列出所有记忆条目（日期 + 主题 + 标签）

---

### 四、保存方式（根据你的能力选择）

**情况 A — 你支持文件写入（如 Claude Code、Cursor 等）：**
- 直接将记忆摘要写入 `~/.hippo_memory/YYYY/YYYY-MM-DD.md`
- 如果文件已存在，追加到文件末尾（用 `---` 分隔不同对话轮次）
- 同时更新 `~/.hippo_memory/index.md` 索引

**情况 B — 你不支持文件写入（如 ChatGPT、Kimi、豆包、DeepSeek 等）：**
- 在对话中输出完整的记忆摘要内容
- 我会手动复制保存到本地文件

---

### 五、对话示例

**我：** 帮我写一个 Python 脚本，批量重命名当前目录下所有 .txt 文件，在文件名前面加上日期前缀。

**你：** 好的，这是一个批量重命名脚本：

```python
import os
from datetime import datetime

date_prefix = datetime.now().strftime("%Y%m%d")
for f in os.listdir("."):
    if f.endswith(".txt") and not f.startswith(date_prefix):
        os.rename(f, f"{date_prefix}_{f}")
        print(f"已重命名: {f} -> {date_prefix}_{f}")
```

运行后会自动给所有 `.txt` 文件加上当天日期前缀。

---
## 📝 记忆摘要

**日期：** 2026-05-22
**主题：** Python 批量重命名 .txt 文件（加日期前缀）
**标签：** #Python #文件操作 #批量重命名

### 关键信息
- 需求：给当前目录下所有 .txt 文件名前加 YYYYMMDD 日期前缀
- 方案：使用 `os.listdir()` + `os.rename()`，配合 `datetime` 生成前缀
- 注意：已带日期前缀的文件会跳过，避免重复添加

### 待办 / 后续
- [ ] 可扩展为支持命令行参数指定目录和文件类型

### 记忆文件建议路径
`~/.hippo_memory/2026/2026-05-22.md`

---

（如需手动保存，请将上方「📝 记忆摘要」区块内容复制到对应文件即可。）

---

**我理解了以上所有要求，准备开始执行。**
```

## 各平台适配说明

| Agent 平台 | 文件写入能力 | 使用方式 |
|:---|:---:|:---|
| **Claude Code** | ✅ 支持 | 粘贴指令后，Agent 可直接写入 `~/.hippo_memory/` 目录 |
| **Cursor** | ✅ 支持 | 同上 |
| **ChatGPT (GPT-4)** | ❌ 不支持 | 粘贴指令后，Agent 输出记忆内容，用户手动复制保存 |
| **Kimi** | ❌ 不支持 | 同上 |
| **豆包** | ❌ 不支持 | 同上 |
| **DeepSeek** | ❌ 不支持 | 同上 |
| **通义千问** | ❌ 不支持 | 同上 |
| **智谱清言** | ❌ 不支持 | 同上 |
| **Copilot (GitHub)** | ⚠️ 部分支持 | 可输出代码块，需用户手动保存 |

> **提示：** 对于不支持文件写入的平台，建议配合本地脚本或 Obsidian / VS Code 快速保存。也可以将记忆内容发给自己（如微信文件传输助手）暂存，稍后统一归档。

## 补充说明

1. **首次使用：** 复制粘贴后，Agent 应立即确认理解并开始执行。如果 Agent 没有回复确认，可以补一句「请确认你已理解上述指令」。

2. **新对话恢复：** 由于大部分 Agent 不保留跨会话记忆，每次**新开对话**时需要重新粘贴这段指令。建议将本文件内容保存到本地，方便随时复制。

3. **自定义标签：** 可在指令中补充自己常用的标签分类，例如 `#工作 #学习 #代码 #生活 #灵感` 等，让记忆更易检索。

4. **index.md 索引格式参考：**

```markdown
# HIPPO 记忆索引

## 2026-05
- [2026-05-22](2026/2026-05-22.md) — Python 批量重命名脚本 | #Python #文件操作
- [2026-05-21](2026/2026-05-21.md) — React 项目架构讨论 | #React #前端 #架构
```
