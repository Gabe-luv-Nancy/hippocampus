## 一、本次已完成操作记录

### 1.1 安装 Nginx

```bash
yum install -y nginx
systemctl start nginx
systemctl enable nginx
```

### 1.2 创建网站目录结构

```
/var/www/html/                      ← 原始位置（已被移动）
/root/clabin_sync/HIPPO/WEBSITE/www/html/  ← 新位置（Syncthing 同步）
```

### 1.3 移动网站文件到 CLABIN 同步目录

```bash
mv /var/www/html /root/clabin_sync/HIPPO/WEBSITE/www/html
```

### 1.4 修改 Nginx 配置

**配置文件位置：** `/etc/nginx/conf.d/clabin.conf`

**修改内容：**
```nginx
# 修改前
root /var/www/html;
alias /var/www/html/assets/;
alias /var/www/html/downloads/;

# 修改后
root /root/clabin_sync/HIPPO/WEBSITE/www/html;
alias /root/clabin_sync/HIPPO/WEBSITE/www/html/assets/;
alias /root/clabin_sync/HIPPO/WEBSITE/www/html/downloads/;
```

### 1.5 修复权限问题

```bash
# nginx worker 进程以 nginx 用户运行，无权限读取 /root/ 下的文件
# 解决方案：给路径上的目录加执行权限，文件加读权限
chmod o+x /root
chmod -R o+r /root/clabin_sync/HIPPO/WEBSITE/www/html/
```

### 1.6 重载配置

```bash
nginx -t                    # 测试配置
nginx -s reload            # 重载配置（不中断服务）
```

## 二、Nginx 启停改查全套操作

### 2.1 基本命令

| 操作 | 命令 |
|------|------|
| 启动 Nginx | `nginx` 或 `systemctl start nginx` |
| 停止 Nginx | `nginx -s stop` 或 `systemctl stop nginx` |
| 重载配置（不中断服务）| `nginx -s reload` |
| 强制停止进程 | `killall nginx` |
| 查看 Nginx 进程状态 | `ps aux \| grep nginx` |
| 测试配置文件是否正确 | `nginx -t` |
| 查看访问日志 | `tail -f /var/log/nginx/clabin_access.log` |
| 查看错误日志 | `tail -f /var/log/nginx/error.log` |

### 2.2 修改配置的正确步骤

```
1. 测试配置是否正确
   nginx -t
   ↓ 看到 "syntax is ok" 才往下走

2. 重载配置（不中断服务）
   nginx -s reload

3. 验证服务正常
   curl http://localhost/
```

### 2.3 编辑配置文件

```bash
# 方式1：vi/vim 编辑
vi /etc/nginx/conf.d/clabin.conf

# 方式2：sed 直接替换（不需要打开文件）
sed -i 's/旧内容/新内容/g' /etc/nginx/conf.d/clabin.conf
```

### 2.4 应急回滚

```bash
# 1. 停掉 nginx
nginx -s stop

# 2. 恢复配置文件（如果有备份）
cp /备份路径/clabin.conf /etc/nginx/conf.d/clabin.conf

# 3. 重新启动
nginx
```

## 三、网站文件位置与编辑方法

### 3.1 当前文件结构

```
/root/clabin_sync/HIPPO/WEBSITE/www/html/
├── index.html                    ← 首页（主要编辑这里）
├── about/
│   └── index.html               ← 关于页
├── downloads/
│   ├── index.html              ← 下载列表页
│   └── sample-file.zip          ← 占位下载文件（替换为真实文件）
└── assets/                      ← 静态资源（css/js/图片）
    ├── css/
    ├── js/
    └── images/
```

### 3.2 Nginx 配置文件位置

```
/etc/nginx/conf.d/clabin.conf     ← 修改 root 路径、端口等
/etc/nginx/nginx.conf             ← 全局配置（一般不改）
```

### 3.3 如何编辑网站内容

**方法 1：在云服务器上直接编辑（VNC 或 SSH）**

```bash
# 编辑首页
vi /root/clabin_sync/HIPPO/WEBSITE/www/html/index.html

# 编辑下载页
vi /root/clabin_sync/HIPPO/WEBSITE/www/html/downloads/index.html
```

**方法 2：在本地编辑，Syncthing 自动同步到云端（推荐）**

```
本地电脑 → 编辑文件 → Syncthing 自动同步 → 云端自动更新
```

文件在云端的路径：`/root/clabin_sync/HIPPO/WEBSITE/www/html/`
对应你本地的 Syncthing 同步文件夹（CLABIN），在 Windows 上大概是：
`X:\CLABIN\HIPPO\WEBSITE\www\html\`

### 3.4 如何添加下载文件

**Step 1：** 把真实文件放入下载目录

```bash
# 在云服务器上
cp /真实文件路径 /root/clabin_sync/HIPPO/WEBSITE/www/html/downloads/

# 或在本地放入 X:\CLABIN\HIPPO\WEBSITE\www\html\downloads\
# Syncthing 会自动同步到云端
```

**Step 2：** 修改下载列表页 `downloads/index.html`，添加下载链接

## 四、SSH 协议与远程连接配置

### 4.1 什么是 SSH

**SSH（Secure Shell）：** 一种加密协议，用于从本地电脑安全地远程连接云服务器。

```
本地 Terminal → 加密连接 → 云服务器（81.70.200.211）
```

### 4.2 SSH 连接方式

#### 方式 A：密码登录（不推荐，不安全）

```bash
ssh root@81.70.200.211
# 然后输入密码
```

#### 方式 B：密钥登录（推荐，安全）

**原理：**
```
本地有私钥（~/.ssh/id_ed25519）
云端有公钥（~/.ssh/authorized_keys）
配对成功即可登录，无需密码
```

### 4.3 当前云服务器 SSH 配置

**状态：** 已有 authorized_keys，但本地私钥路径未知

**需要确认：**
- 你本地电脑（Windows WSL）的 SSH 私钥在哪个路径？
- 私钥文件名称是什么？（id_ed25519？id_rsa？其他？）

### 4.4 查看云端已授权的公钥

```bash
cat ~/.ssh/authorized_keys
```

### 4.5 如何配置从本地 Windows WSL 免密码 SSH

**Step 1：** 在本地 WSL 查看是否有密钥

```bash
ls -la ~/.ssh/
```

**Step 2：** 如果没有，生成密钥

```bash
ssh-keygen -t ed25519 -C "Gabe@TencentCloud"
# 一路回车，密钥默认保存在 ~/.ssh/id_ed25519
```

**Step 3：** 把公钥添加到云端

```bash
# 方式1：手动复制（本地执行）
cat ~/.ssh/id_ed25519.pub
# 复制输出的内容

# 然后在云服务器上执行
echo "粘贴的公钥内容" >> ~/.ssh/authorized_keys

# 方式2：ssh-copy-id（本地执行）
ssh-copy-id -i ~/.ssh/id_ed25519.pub root@81.70.200.211
```

**Step 4：** 测试连接

```bash
ssh -i ~/.ssh/id_ed25519 root@81.70.200.211
```
